# AI-агенты в метамодели

## Статус в метамодели
AI-агент = **label на component** (type=agent). Не отдельная entity kind. MCP-агент аналогично (type=mcp).

## Требования
- Каждый агент должен иметь **собственный service account** с ограниченными правами
- Учёт как component + business_role (двойной маппинг)

## AI-агенты как потребители метамодели
Стратегическое направление: метамодель должна быть **машиночитаемой** для AI-агентов.

**Use-cases:**
- Семантический поиск: «рублёвый платёж» → объект → таблицы → системы
- Text2SQL: разметка колонок (категориальные типы, диапазоны значений, примеры запросов)
- «Что если» аналитика: «что будет с кредитами при повышении ключевой ставки на 2%»
- Виртуальный архитектор: агент, видящий весь стек от бизнес-процессов до компонентов

## MCP-интерфейс
Метамодель предоставляет **MCP-интерфейс** для программного доступа к сущностям. MCP каталога готов для финансов/ритейла.

## Связанные entity_kinds
`component`, `business_role`, `data_attribute`, `business_entity`, `api`
