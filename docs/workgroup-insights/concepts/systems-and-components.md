# Systems & Components (Системы и компоненты)

## Определение (по состоянию на 2025-12-09)

ИТ-слой метамодели: как связаны информационные системы, компоненты, интеграции и ресурсы.

## Ключевые сущности и их определения

### IT System (Информационная система)
- Группа компонентов, объединённых под одной системой
- Может быть монолитом (система = один компонент)
- Компонент может «вырасти» в систему при достаточном масштабе

### Component (Компонент)
- Микросервис, библиотека, сервис — то, что команды пишут и деплоят
- Центральная единица application layer
- Регистрируется в CMDB

### Integration (Интеграция)
- **Определение:** взаимодействие двух компонентов при наличии продюсера И потребителя
- **Сужение:** простое подключение к Kafka — не интеграция
- Реализует информационные потоки
- Спецификация (что публикует продюсер) может существовать без интеграции

### Logical Resource (Ресурс)
- Application layer: то, от чего зависит компонент
- Примеры: Kafka topic, S3 bucket
- **Не относится к инфраструктурному слою**

### Infra Resource (Инфраструктурный ресурс)
- Кластер Kafka, брокеры, worker nodes Kubernetes
- Отдельный слой ниже application layer

## Связанные entity_kinds
`it_system`, `component`, `functional_subsystem`, `integration`, `information_flow`, `api`, `logical_resource`, `infra_resource`, `vendor_product`, `software_component`, `technology`, `source`

## Примеры
- Apache Kafka (кластер = инфра, топик = ресурс, брокер = инфра-инстанс)
- Kubernetes (worker node — инфраструктура)
- REST API, MQ, FTP/SFTP — типы интеграций
- CMDB — мастер-система для компонентов
