# Модель владения данными (Ownership)

## Принципы (консолидированные)

1. **Владелец = организационная единица**, не физлицо (при увольнении — остаётся за командой)
2. **Owner дата-продукта приоритетнее** owner дата-объекта
3. Объект без продукта = **сам является дата-продуктом** (схлопывание)
4. Владелец на уровне дата-объекта **наследуется** от дата-продукта (отдельный не нужен)
5. **Замещающий** (deputy) — метрика качества продукта; его отсутствие снижает рейтинг

## Многоуровневая модель
```
Бизнес-владелец (Product Owner)
  └→ Технический ответственный (Data Engineer lead)
       └→ Data Steward (качество, методология)
            └→ Группа поддержки (SLA-специфичные контакты)
```

## Механизм через SLA-роли
В пилоте Data NARAP: владение выстраивается через **наборы SLA** — каждый контракт включает разные типы SLA для разных аспектов (технических, бизнесовых, инцидентных).

## Открытый контракт
Поставщик публично декларирует условия без конкретного потребителя-подписанта. Расширение SLA для нового потребителя = расширение для всех.

## Связанные entity_kinds
`data_product`, `data_contract`, `organizational_unit`, `business_role`, `sla`
