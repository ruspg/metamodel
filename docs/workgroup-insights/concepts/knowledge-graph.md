# Knowledge Graph (Граф знаний)

## Определение (2026-03-26)

Граф знаний ≠ метамодель. **Метамодель = схема** (типы сущностей и связей). **Граф знаний = наполненный данными экземпляр**, построенный из существующих систем банка.

## Источники данных для графа
- CAP (граф компонентов через трейсы/спаны)
- Инфраструктурный каталог (CMDB/7DB)
- Дата-линейдж (DataHub, Airmon)
- Реестры бизнес-процессов (CAP, Ops Process Tool)
- Entity Store / Антистор (бизнес-сущности)

## Технологический стек
- **Формат метамодели**: OWL/Turtle (.ttl) — конвертируется из YAML через Python
- **Хранение**: Git (YAML) → Postgres (до ~1000 объектов) → **Neo4j** (целевое, десятки/сотни тысяч)
- **Доступ**: MCP-серверы поверх OWL — агенты делают запросы по графу (связи 1-го, 2-го уровня глубины)

## Практические use-cases
- **BPMN-генерация**: MCP метамодели + BPMN MCP → автогенерация из транскриптов/Confluence/Excel (первый кандидат: команда Антифрод Operations)
- **Семантический поиск**: «рублёвый платёж» → бизнес-объект → таблицы → системы
- **«Что если» аналитика**: сценарное моделирование на графе
- **Контекст для AI-агентов**: машиночитаемая онтология через MCP

## Связанные entity_kinds
Все — граф знаний объединяет все уровни метамодели.
