# Сертификация дата-продуктов

## Определение
Система уровней зрелости/соответствия дата-продукта, привязанная к типу потребления и банковским стандартам. Альтернатива жёстким обязательным полям — «данные без стандарта лучше, чем их отсутствие».

## Три категории потребления (2026-03-25)
1. **Отчётность/регуляторика** — высший уровень (ЦБ, ФНС, ЕЦБ)
2. **Аналитика** — средний уровень
3. **Операционка (real-time OLT-P)** — базовый уровень

## Сертификация vs уровень зрелости DoA
- **DoA** = техническое качество (минимальный контракт, SLI)
- **Сертификация** = бизнес-соответствие + банковская методология

## Три уровня зрелости контракта (2026-02-25)
- **L1** — только владелец (матрица эскалации, без SLI)
- **L2** — описание + разбивка на продукты
- **L3** — полный (SLI, разбивка, поддержка)

## Механизм
- Автоматический **отзыв сертификата** при нарушении SLA
- Сертификат может отзываться и вручную
- Референс: все крупнейшие банки с Data Mesh делают сертификацию

## Связанные entity_kinds
`data_product`, `data_contract`, `requirement`, `sla`, `metric`
